// architecture

Six agents. One typed state.

A LangGraph StateGraph passes one shared state down the chain. Click any node to see what it does — note only two of the six call the LLM.

01Intake agentLLMClassifies every ticket by category, urgency and sentiment.+
Outputs category / urgency / sentiment at temperature 0 for determinism. Includes fallback JSON parsing for local-model quirks.
02Routing agentRulesDeterministically assigns the ticket to the right team.+
Pure logic, zero LLM calls. Maps category → team (Finance / Engineering / Logistics / Account) from a config dict.
03Retrieval agentRAGPulls the top-3 knowledge-base chunks, filtered by category.+
all-MiniLM-L6-v2 embeddings over ChromaDB. Category-aware filtering lifted hit-rate from 78% → 92%. No API calls.
04Drafting agentLLMWrites an empathetic reply grounded strictly in retrieved chunks.+
Temperature 0.3 for slight variation. The prompt forbids inventing information and leads with an apology on angry sentiment.
05Escalation agentRulesFlags tickets that need a human.+
Three triggers: urgency in {high, critical}, sentiment == angry, or zero KB matches. Saves the reason for the reviewer.
06Follow-up agentRulesSchedules a follow-up on any escalated or angry ticket.+
Sets followup_required = true. In production this fires a Celery task or Slack alert after 24h.
